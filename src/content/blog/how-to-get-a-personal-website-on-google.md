Getting a personal website on Google is not a magic submission trick.

Google needs to discover the URLs, crawl them, understand the pages, and decide whether they are useful enough to include and show for relevant searches. A small personal site can do all the right technical basics and still need time.

The goal is to make discovery easy and give the site enough useful structure to deserve indexing.

## The short version

Do this first:

1. Make sure the site is live on a public URL.
2. Make sure important pages are not blocked by `noindex`, password protection, or broken routing.
3. Publish a clear homepage, about page, contact path, and a few useful pages.
4. Link important pages together from the homepage, navigation, topic pages, or related reading sections.
5. Submit the sitemap in Google Search Console.
6. Use URL Inspection for a few important URLs.
7. Wait, monitor, and improve the pages instead of repeatedly resubmitting them.

Google's own recrawl guidance says crawling can take from days to weeks, and requesting a crawl does not guarantee immediate inclusion. The useful response is patience plus better pages, not panic.

## 1. Make the site crawlable

Before thinking about Search Console, check the basics.

Your important pages should:

- return a normal `200` response;
- be reachable from links;
- not require login;
- not use `noindex`;
- not be blocked in `robots.txt`;
- have clean, stable URLs;
- load enough text for the page to be understood.

Many personal websites fail because the author spends time on content but misses a basic visibility problem.

If the site is still pre-launch, use the [personal website launch checklist](/blog/personal-website-launch-checklist/) before submitting anything.

## 2. Publish pages worth discovering

Indexing is not only a technical issue.

A new personal website with one vague homepage gives Google and readers very little to understand.

At minimum, the site should have:

- a homepage that explains the site;
- an about page that builds trust;
- a contact path if contact is relevant;
- one main section such as Writing, Work, Projects, Blog, or Guides;
- a few useful pages that answer specific questions or show specific work.

If the page set is unclear, read [what pages should a personal website have](/blog/what-pages-should-a-personal-website-have/). If the homepage is vague, use [what should a personal website homepage say](/blog/what-should-a-personal-website-homepage-say/).

## 3. Create a sitemap

A sitemap helps Google discover URLs on your site, especially when the site is new or has many pages.

For a small personal website, the sitemap should usually include:

- homepage;
- main pages;
- blog posts or guide pages;
- topic or category pages if they are useful;
- updated URLs when important pages change.

Google's sitemap documentation describes sitemap submission as a discovery signal, not a guarantee that every URL will be crawled or indexed. That distinction matters. A sitemap helps Google find pages. It does not make weak pages strong.

This site, for example, exposes its sitemap index at `/sitemap-index.xml`. A personal website should have something similarly easy to find.

## 4. Set up Google Search Console

Google Search Console is the place to verify ownership, submit the sitemap, inspect URLs, and monitor indexing and performance.

For a personal website, the setup is usually simple:

1. Add the domain or URL-prefix property.
2. Verify ownership.
3. Submit the sitemap.
4. Inspect the homepage.
5. Inspect a few important articles or pages.
6. Check indexing and performance over time.

Do not inspect every page every day. Search Console is a diagnostic tool, not a ritual.

For the operational version, read [Google Search Console for personal websites](/blog/google-search-console-for-personal-websites/).

## 5. Use URL Inspection for important URLs

The URL Inspection tool can show what Google knows about a specific page and whether the live page appears indexable.

Use it for:

- the homepage;
- a new cornerstone guide;
- a page that should be indexed but is missing;
- a page you materially updated;
- a page with a suspected canonical, noindex, or crawl issue.

Google's support documentation notes that requesting indexing can be used for a URL, but there are limits and no guarantee of inclusion. For many pages, a sitemap is the better route.

The practical rule:

- use URL Inspection for a few important URLs;
- use the sitemap for many URLs;
- improve the page if indexing does not happen.

## 6. Link the site together

Google discovers pages through links. Readers do too.

A personal website should not depend only on the sitemap. Important pages should be reachable from the site itself.

Check whether:

- the homepage links to the best starting pages;
- the about page links to important work or writing;
- blog posts link to related guides;
- topic pages list and explain clusters;
- older pages link to newer stronger pages.

If a page is orphaned, it may still appear in a sitemap, but it is weak inside the site.

Use [internal linking for bloggers](/blog/internal-linking-for-bloggers/) to turn isolated articles into a connected archive.

## 7. Search for the site carefully

After launch, people often search their name or domain and panic.

Use several checks:

- search the exact domain;
- search the exact title of the homepage;
- search `site:yourdomain.com`;
- check Search Console indexing reports;
- inspect the specific URL.

If the site is very new, the most common answer may simply be time. Google's crawling and indexing FAQ notes that newness is a common reason a site is not indexed yet.

## 8. Do not keep resubmitting the same URL

Repeatedly requesting indexing for the same page does not force Google to rank it.

If a page is not getting indexed or shown, improve the underlying issue:

- make the page more useful;
- improve title and description alignment;
- add internal links;
- remove duplicate or thin pages;
- fix crawl or canonical problems;
- make the site easier to understand.

This is where [personal website SEO checklist](/blog/personal-website-seo-checklist/) and [personal website audit checklist](/blog/personal-website-audit-checklist/) matter more than another submission click.

## 9. What if the site still does not show up?

Work through this order:

1. Confirm the site is public.
2. Confirm the URL is not blocked or noindexed.
3. Confirm the canonical URL is the one you expect.
4. Submit the sitemap.
5. Inspect the homepage.
6. Add internal links to important pages.
7. Improve thin or vague pages.
8. Wait a reasonable amount of time.

If the homepage is indexed but deeper pages are not, the issue may be site structure or page quality. If nothing is indexed, check technical visibility first.

## The standard

The goal is not only to get a page into Google's index.

The goal is to build a personal website that is clear enough for Google to discover and useful enough for people to stay, read, and continue.

Submission helps discovery. Structure and content create value.
