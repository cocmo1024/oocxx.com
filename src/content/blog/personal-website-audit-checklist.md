A personal website audit should not begin with advanced SEO tools.

It should begin with a simpler question:

**Can a real visitor understand and use this website without already knowing you?**

If the answer is no, publishing more pages will not solve the core problem. More content only helps when the site already has enough structure, trust, and readability to support it.

## 1. Audit the homepage first

Open the homepage and ask:

- Can a stranger understand what the site is about in ten seconds?
- Does the first screen say who or what the site represents?
- Is there a clear next step?
- Are the main sections visible without guessing?
- Does the homepage route people to the best pages?

A weak homepage makes every other page work harder.

If the top of the page is vague, use [what should a personal website homepage say](/blog/what-should-a-personal-website-homepage-say/) and the [personal website homepage checklist](/blog/personal-website-homepage-checklist/) before changing the rest of the site.

## 2. Audit the core page set

Most personal websites need a small set of durable pages:

- Home;
- About;
- Work, Projects, Writing, or Guides;
- Contact;
- topic or archive pages if the site publishes regularly.

The audit question is not whether every site has the same pages. It is whether the site gives readers enough context to understand the person, the work, and the next useful path.

If the page set is unclear, use [what pages should a personal website have](/blog/what-pages-should-a-personal-website-have/) as the baseline.

## 3. Audit navigation labels

Navigation should be boring in the best way.

Check whether the top-level links use labels people understand:

- About;
- Work;
- Projects;
- Blog;
- Guides;
- Contact.

Labels such as Lab, Notes, Dispatches, Experiments, or Playground can work after the site is already clear. They are risky when they replace basic orientation.

Also check whether the menu is too large. A small site usually does not need ten top-level links.

Use [website navigation best practices](/blog/website-navigation-best-practices/) if the menu feels clever but unclear.

## 4. Audit the about page

The about page should answer:

- who is behind the site;
- what the person or project does;
- why the subject is credible;
- what readers should explore next;
- how to contact or verify the person if needed.

A weak about page is not only a branding problem. It is a trust problem.

Do not make it a full autobiography unless the life story is the point of the site. Most readers need context, credibility, and direction.

Use [how to write an about page](/blog/how-to-write-an-about-page/) if the page is either too vague or too long.

## 5. Audit content depth

Open your most important articles and ask:

- Does this page answer one clear question?
- Does it give enough detail to help someone act?
- Does it include examples, criteria, or tradeoffs?
- Does it link to the next useful page?
- Would the page still be useful if the reader came directly from search?

Thin content is not only short content. A long page can still be thin if it repeats obvious advice without helping the reader make a decision.

If many posts feel disconnected, use [blog categories vs tags](/blog/blog-categories-vs-tags/) and [internal linking for bloggers](/blog/internal-linking-for-bloggers/) to rebuild the system.

## 6. Audit internal links

Every important page should help readers continue.

Check whether:

- beginner pages link to deeper pages;
- detail pages link back to broader guides;
- topic clusters are connected;
- old posts link to newer stronger posts;
- related reading sections are genuinely related.

Internal links should not exist only for search engines. They should make the site easier to read.

If a reader lands on one article and has no obvious next step, the site is leaking attention.

## 7. Audit titles and descriptions

Every indexable page should have a title and description that match the visible content.

Check:

- Does the title describe the real topic?
- Does the description summarize the actual page?
- Does the page deliver what the title promises?
- Are multiple pages targeting the same question by accident?

If the title promises a checklist, the page should include a checklist. If the description promises examples, the page should include examples. This sounds basic, but many small sites drift away from their own metadata.

For the SEO-specific version, use the [personal website SEO checklist](/blog/personal-website-seo-checklist/).

## 8. Audit mobile reading

Mobile quality matters because many readers will first encounter the site on a phone.

Check:

- font size;
- line length;
- spacing between links;
- menu behavior;
- image scaling;
- whether the first screen is understandable;
- whether long paragraphs become tiring.

A personal website does not need a complex mobile interface. It needs to be comfortable to read.

## 9. Audit publishing direction

Before publishing more posts, ask whether the site has a visible direction.

Good signs:

- several pages support the same subject;
- categories are few and stable;
- tags describe real recurring ideas;
- the homepage points to the best starting pages;
- old posts are updated when better guides exist.

Weak signs:

- random topics with no connection;
- many categories with one post each;
- old posts that contradict newer advice;
- a blog added only because the template included one.

If you are unsure whether the blog belongs at all, read [should a personal website have a blog](/blog/should-a-personal-website-have-a-blog/).

## 10. Decide what to fix first

Use this order:

1. Fix homepage clarity.
2. Fix the core page set.
3. Fix navigation.
4. Improve the about and contact paths.
5. Strengthen the best existing articles.
6. Add internal links.
7. Publish new pages only where a real gap remains.

That order prevents the common mistake: adding more content to a site that still lacks basic orientation.

## The audit standard

A strong personal website is not the biggest site.

It is the site where a stranger can arrive, understand the purpose, trust the source, read comfortably, and find the next useful page.

That is the standard worth auditing against.
